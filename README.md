# Открытки — версия по новому ТЗ

## Что внутри
- `server.js` — локальный агент: HTTP + WebSocket + автоматическое обнаружение координатора по сети.
- `public/` — HTML-приложение и popup-окно.
- `Запуск приложения.vbs` — запуск в один клик без командной строки.
- `Запуск приложения.cmd` — резервный запуск.

## Как запускать на Windows
1. Распаковать одну и ту же папку на каждый из 4 ПК.
2. На каждом ПК открыть `Запуск приложения.vbs`.
3. В меню выбрать роль ПК-1/ПК-2/ПК-3/ПК-4.
4. На одном ПК включить `Сервер: Да`.
5. На остальных ПК этот флаг заблокируется автоматически.

## Что реализовано
- Сцена 3440x1440 с 8 открытками в строгой сетке.
- Клики по лицу и обороту, бесконечное переключение.
- 16 последовательных звуков для каждого ПК, предыдущий звук прерывается.
- Сценарий запускается на 17-м клике или по MIDI-ноте запуска.
- Начальная последовательность popup-окон: ПК1 → ПК2 → ПК3 → ПК4 → ПК3 → ПК2.
- Затем окно автоматически удерживается 2 минуты на ПК1, потом 2 минуты на ПК2, потом 2 минуты на ПК3.
- После этого окно открывается на ПК4 и закрывается той же MIDI-нотой, что запускает сценарий.
- Popup содержит 8 разворотов: слева лицо + текст, справа оборот + текст.
- Режим редактирования текста: клавиша `E` или `У`.
- Меню скрывается на `M`, fullscreen на `F`, сброс настроек на `Shift+R`.
- Автообнаружение координатора по локальной сети через UDP broadcast.

## Техническое ограничение
Для полностью самодостаточного запуска без установленного Node.js нужен отдельный Windows-бандл с встроенным runtime. В этой сборке сохранён формат HTML + Node.js агент; launcher скрывает командную строку, но `node.exe` должен быть установлен в системе и доступен в PATH.


Синхронизация между ПК:
- На серверном ПК включить "Сервер: Да".
- На остальных ПК ввести IP серверного ПК в поле "IP серверного ПК" и не включать режим сервера.
- Все клики и сценарий идут только через серверный ПК.


## PDF для каждого ПК

Положите PDF-файлы в папку `public/pdfs/`. В меню приложения можно выбрать отдельный PDF для ПК-1, ПК-2, ПК-3 и ПК-4 и сохранить настройку на серверном ПК. По умолчанию используются `pc1.pdf`, `pc2.pdf`, `pc3.pdf`, `pc4.pdf`.
